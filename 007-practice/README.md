**Краткое описание: что делает Redis и зачем нужен TTL**

`Redis` - это высокопроизводительная `in-memory` система хранения данных, которая работает по принципу "ключ-значение"

`Redis` применяется там, где важна **скорость** и **масштабируемость**

- **Кэширование результатов** запросов к базе данных

- **Хранение сессий в веб-приложения**

- **Очереди задач**

- **Счетчикик и рейтинги**
----------

`TTL` **(Time-To-Live)** - время жизни ключа

Если TTL установлен, то ключ автомаитчески иссчезает из `Redis` после указанного времени

Удобно для кэширования: данные живут в кжше огранниченное время и автоматически обновляются

`TTL` помагает избежать устаревших данных и перегрузки памяти
